# Fast Track Plan: Mini E-Commerce System

## 3 Sprints per Day (2 Hours Each)

Total Duration: ~8 Days

# Overall Strategy

Each day follows:

- **Sprint 1 (2 hrs): Backend development**
- **Sprint 2 (2 hrs): Frontend development**
- **Sprint 3 (2 hrs): Integration + testing + fixes**

This ensures:

- No feature stays “half-built”
- Every day produces working output
- Constant integration avoids backlog issues

# Day 1: Foundation + Auth Core

## Sprint 1 (Backend Setup)

- Initialize Node + Express project
- Setup folder structure
- Setup MySQL connection
- Create base server + environment config

## Sprint 2 (Frontend Setup)

- Initialize React project
- Setup routing structure
- Setup folder architecture (pages/components/services)

## Sprint 3 (Integration Setup)

- Test backend API health route
- Connect frontend to backend
- Confirm DB connection working

# Why This Structure Works

This is important:

- Every day produces **working increments**
- Backend + frontend + integration happen daily (no drift)

- Mimics real agile sprint cycles used in companies
